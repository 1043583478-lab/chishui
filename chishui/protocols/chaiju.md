# 4 步拆局工作法 (Chaiju Protocol)

> 定位: 应用框架。当你面对的是一个复杂、多方、有历史的"结构题"时使用。

## 总纲

这不是"你一句，我输出八段"的技能，而是一套四步工作法：

> **先问清，再拆局；先定线，再交付。**

```
┌────────────┐    ┌────────────┐    ┌────────────┐    ┌────────────┐
│ 01 先问清   │───▶│ 02 再拆局   │───▶│ 03 后定路线 │───▶│ 04 再交付   │
│            │    │            │    │            │    │            │
│ 目标/事件   │    │ 矛盾/阶段   │    │ 路线/顺序   │    │ 文字分析    │
│ 人物/尝试   │    │ 力量/风险   │    │ 边界/动作   │    │ HTML 报告  │
│ 约束       │    │            │    │            │    │            │
└────────────┘    └────────────┘    └────────────┘    └────────────┘
```

---

## 第一步：先问清（澄清阶段）

### 目标
在进入任何分析之前，确保关键结构不缺位。不是在"收集信息"，是在"补齐结构"。

### 核心操作

1. **第一轮优先锁目标**
   - 不问"你的问题是什么"，问"你最想推进的结果是什么"
   - 目标不是问题分类，而是行动的北极星

2. **补齐五个关键结构位**
   - 目标：你最想推进的结果
   - 事件：最近一次最能说明问题的关键事件
   - 人物：关键人物、第三方、关系人
   - 尝试：你已经做过的动作
   - 约束：真正的限制、底线和代价

3. **澄清使用选项式追问**
   - 不用开放式问题（"你再详细说说"）
   - 用带 A/B/C/D/其他 的选项式追问帮助用户聚焦
   - 如果用户给了聊天记录/微信片段等局部措辞，先把它们当证据材料，不下沉成"帮你写一句怎么发"

4. **澄清未闭合前，不进入分析**
   - 关键结构缺位时，不做"我先粗略分析一下"
   - 明确告诉用户：还需要哪些信息才能开始分析

### 完成标准
- 目标清晰（可验证的"完成"定义）
- 关键事件被锚定（有时间、有具体细节）
- 人物关系被勾勒（谁是 decision maker、谁被影响）
- 约束被明确（什么是不能动的）

---

## 第二步：再拆局（分析阶段）

### 目标
先看清局面怎么卡住的，再判断真正的主问题。不是"列出所有问题"，是"找出主要矛盾"。

### 核心操作

1. **矛盾分析**（调用 `contradiction-analysis`）
   - 列出局面中所有的对立统一对
   - 找出主要矛盾——"由于它的存在和发展规定或影响着其他矛盾"
   - 分析每个矛盾的特殊性——是资源性的？结构性的？阶段性的？

2. **阶段判断**（调用 `protracted-war-strategy` 的阶段识别）
   - 用户处于哪个阶段：防御（学习和生存）、相持（拉锯）、还是反攻（可以主动出击）？
   - 哪些因素随时间增强？哪些随时间衰减？
   - **元判断**：用户处于"战争模式"还是"耕种模式"？

3. **力量分析**（调用 `concentrate-forces` + `mass-line`）
   - 用户手里有什么牌？哪些是可以集中的力量？
   - 支持者/同盟在哪里？哪些是可以争取的中间力量？

4. **风险识别**
   - 最大的风险源在哪？（不是"可能出错"的泛泛列表）
   - 如果主要矛盾判断错了，后果是什么？

### 完成标准
- 主要矛盾被命名并解释（为什么这个是主要的）
- 阶段被判定（防御/相持/反攻，or 耕种模式）
- 关键力量的分布被画出来
- 核心风险被锚定

---

## 第三步：后定路线（路线设计）

### 目标
不是空讲方向，而是排出先后手和边界线。

### 核心操作

1. **主路线**
   - 基于主要矛盾和当前阶段，推荐一条主路线
   - 路线必须有：方向（往哪走）、顺序（先做什么后做什么）、支点（关键撬动点在哪）

2. **路线比较**
   - 不只给一条路——给 2-3 条可选路线
   - 每条标注：成立前提、所需资源、核心风险、适用场景
   - 推荐一条并说明为什么

3. **边界线**
   - 什么情况下这条路线不可行？
   - 什么信号表明需要切换路线？
   - 什么是绝对不能做的？

4. **下一步动作**
   - 给出近期（1-2 周内）的 3-5 个具体动作
   - 每个动作带观察点：做完后看什么信号判断方向对不对

### 完成标准
- 有一条明确的主路线
- 有备选路线和切换条件
- 有近期可执行的具体动作
- 有边界线和危险信号

---

## 第四步：再交付（输出阶段）

### 文字版深度分析

最少包含：
- 问题重述（你确认过的版本）
- 核心判断（主要矛盾 + 阶段）
- 推荐路线 + 理由
- 风险提醒
- 下一步动作

### HTML 报告

在文字版基础上增加可选的视觉模块：
- 关系图（谁影响谁、谁卡谁、谁依赖谁）
- 时间线（关键事件的时间序列）
- 路线比较表
- 控制点分布表（名义归属 vs 现实掌握方）
- 证据链（"为什么这样判断"的依据）
- 人物与关系清单

详见 [html-report.md](./html-report.md)

## 与对话协议的区别

| 维度 | 拆局协议 (新青年) | 对话协议 (长征机) |
|------|-------------------|-------------------|
| **目标** | 产出可交付的分析结果 | 通过连环追问帮用户自己想清楚 |
| **节奏** | 四步结构化推进 | 五阶段螺旋深入 |
| **交付** | 文字分析 + HTML 报告 | 对话本身就是交付 |
| **适用** | 用户要"帮我分析这个局面" | 用户要"帮我理清思路" |
| **风格** | 参谋部作战分析 | 导师一对一谈话 |

**选择规则**：如果用户要报告/要对外分享/要存档 → 走拆局协议。如果用户要自己想清楚/要哲学讨论 → 走对话协议。
